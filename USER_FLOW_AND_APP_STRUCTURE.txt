STEP 1: INITIAL APP LAUNCH
---------------------------
Location: app/index.tsx
- User launches the app
- App checks authentication state
- If not authenticated â†’ Redirect to Login/Register
- If authenticated â†’ Check profile completion â†’ Redirect appropriately

STEP 2: USER REGISTRATION
--------------------------
Location: app/(auth)/register.tsx

Input Fields:
- Full Name (required, min 2 characters)
- Email (required, valid email format)
- Password (required, min 6 characters)
- Confirm Password (must match password)

Process:
1. User fills in registration form
2. Client-side validation checks all fields
3. On submit, calls supabase.auth.signUp() with:
   - Email
   - Password
   - User metadata (full_name)
4. Database trigger automatically creates profile record in 'profiles' table
5. Default profile values set:
   - user_type: 'Customer' (default)
   - is_verified: false
   - verification_status: 'Unverified'
   - created_at: current timestamp

Backend (Automatic):
- Supabase Auth creates user in auth.users table
- Database trigger creates profile in profiles table
- Profile linked to auth user via profiles.id = auth.users.id

Success:
- User is authenticated
- Redirect to: app/(auth)/onboarding.tsx

CUSTOMER FLOW - POSTING A JOB
------------------------------
Location: app/(tabs)/post-job.tsx

Access Point:
- Home screen â†’ "+ Post Job" button
- Or from Categories â†’ "Need this service" button

Required Fields:
1. Job Title (text, required)
   Example: "Need help moving furniture"

2. Description (textarea, required)
   Details about the job, requirements, special instructions

3. Category (picker, required)
   - Categories fetched from 'categories' table
   - User selects from predefined list

4. Location (text, required)
   Where the service is needed

5. Execution Date (date picker, required)
   When the job should be done

6. Preferred Time (dropdown, required)
   Options: Morning, Afternoon, Evening, Flexible

7. Budget Range (optional)
   - Min Budget (numeric)
   - Max Budget (numeric)
   - Validation: Max must be > Min

8. Photos (optional)
   - Multi-photo upload
   - Up to 10 photos
   - Used to show examples of work needed

Process Flow:
1. User fills in job details
2. Client-side validation checks required fields
3. On "Post Job" button:
   - Validates all fields
   - Creates job record in 'jobs' table with:
     * customer_id: user's profile ID
     * category_id: selected category
     * title, description, location
     * budget_min, budget_max
     * execution_date_start
     * preferred_time
     * photos: JSON array
     * status: 'Open'
     * expires_at: 30 days from now

4. Database insertion
5. Success notification
6. Providers in area notified (via notification system)
7. Redirect back to previous screen

Database Table: jobs
- customer_id (references profiles.id)
- category_id (references categories.id)
- status: 'Open' | 'InProgress' | 'Completed' | 'Cancelled' | 'Expired'

Job Lifecycle:
Open â†’ Provider submits quote â†’ Customer accepts â†’ InProgress â†’ Completed

PROVIDER FLOW - CREATING A SERVICE LISTING
-------------------------------------------
Location: app/(tabs)/create-listing.tsx

Access Point:
- Home screen â†’ "+ Create Listing" button (for Providers/Hybrid users)
- Profile â†’ "Create New Listing"

Listing Types:
1. SERVICE (Standard service offering)
2. CUSTOM SERVICE (Product/service with custom options, shipping, etc.)

STANDARD SERVICE FIELDS:
------------------------
1. Listing Type (toggle)
   - Service (default)
   - Custom Service

2. Title (text, required)
   Example: "Professional House Cleaning"

3. Description (textarea, required)
   Detailed description of service offered

4. Category (picker, required)
   Service category selection

5. Price Type (radio buttons, required)
   - Hourly Rate
   - Fixed Price

6. Price (numeric, required)
   Base price for the service

7. Duration (numeric, conditional)
   - Only if Hourly Rate selected
   - Estimated hours for the job

8. Photos (optional but recommended)
   - Up to 10 photos
   - Shows examples of work

9. Availability (calendar, required)
   - Select available days
   - At least one day must be selected
   - Days: Monday through Sunday

10. Tags (comma-separated, optional)
    Keywords for search discoverability

CUSTOM SERVICE ADDITIONAL FIELDS:
----------------------------------
11. Fulfillment Type (multi-select, required)
    Options:
    - Local Pickup
    - Shipping
    - Digital Delivery

12. Item Specifications (conditional, required for Shipping):
    - Weight (oz)
    - Dimensions:
      * Length (inches)
      * Width (inches)
      * Height (inches)

13. Fulfillment Window (numeric, required)
    Days needed to complete order (1-30 days)

14. Custom Options (optional)
    - Color variations
    - Size options
    - Material choices
    - Customization text fields

Process Flow:
1. User selects listing type
2. Fills in all required fields
3. If Custom Service:
   - Additional fulfillment options shown
   - Shipping calculator enabled if shipping selected
4. On "Create Listing" button:
   - Validates all fields
   - Creates listing in 'service_listings' table
   - If Custom Service with options:
     * Creates records in 'fulfillment_options' table
     * Creates records in 'custom_product_options' table
   - Uploads photos to storage
   - Creates availability records

5. Success notification
6. Listing goes live immediately (is_active: true)
7. Redirect to provider dashboard

Database Tables:
- service_listings (main listing data)
- fulfillment_options (for custom services)
- custom_product_options (customization choices)
- listing_availability (provider schedule)

Listing Visibility:
- Appears in category browse
- Searchable by keywords
- Shows up in recommendations
- Visible to customers in provider's service area

CUSTOM SERVICE CONSULTATION FLOW:
----------------------------------
For complex custom orders:
1. Customer finds custom service listing
2. Clicks "Request Consultation"
3. Fills out consultation request form:
   - Project description
   - Budget range
   - Timeline
   - Special requirements
4. Provider receives consultation request
5. Provider can:
   - Accept and schedule video call
   - Send detailed quote
   - Decline with reason
6. If accepted â†’ Video consultation scheduled
7. After consultation â†’ Provider sends custom quote
8. Customer accepts quote â†’ Order created
9. Provider fulfills order
10. Customer confirms receipt â†’ Payment released

================================================================================
                        SPECIAL FEATURES & FLOWS
================================================================================

ESCROW SYSTEM
-------------
1. Customer books service â†’ Payment held in escrow
2. Provider completes service
3. Customer confirms completion
4. Payment released to provider (minus fees)
5. If dispute â†’ Admin mediates

RECURRING BOOKINGS
------------------
1. Customer books service
2. Option: "Make this recurring"
3. Select frequency (weekly, bi-weekly, monthly)
4. Set end date or indefinite
5. Auto-booking and charging

PAYMENT PLANS
-------------
1. Customer books expensive service
2. Option: "Pay in installments"
3. Select plan (e.g., 4 payments over 2 months)
4. Auto-charge on schedule
5. Service starts after first payment

CUSTOM PRODUCT FLOW
-------------------
1. Customer finds custom service listing
2. Selects custom options (color, size, etc.)
3. Adds customization text/instructions
4. Requests consultation (if needed)
5. Provider sends custom quote
6. Customer accepts â†’ Order created
7. Provider fulfills â†’ Customer confirms â†’ Payment released

VIDEO CONSULTATIONS
-------------------
1. Customer requests consultation
2. Provider accepts and schedules
3. Both receive meeting link
4. Join video call at scheduled time
5. Discuss project details
6. Provider sends follow-up quote
