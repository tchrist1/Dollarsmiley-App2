IMPLEMENTATION NOTES:

1. Database Structure:
   - All categories and subcategories are stored in the 'categories' table
   - Parent categories have parent_id = NULL
   - Subcategories reference their parent via parent_id
   - Each subcategory has an image_url field with a Pexels image

2. Image Sources:
   - All subcategory images are from Pexels
   - Image URLs are stored directly in the database
   - The description field contains the AI image generation prompt
     (for reference/future regeneration if needed)

3. Global Usage:
   - CategoryPicker components automatically use this data
   - Search and filter systems reference these categories
   - Create Listing, Edit Listing, and Job Posting forms use this structure
   - All category-based navigation uses the database as the source of truth

4. UI Display:
   - Categories are displayed in the left panel (vertical list)
   - Subcategories are shown in the right panel (3-column grid)
   - Subcategory images are displayed in circular containers
   - Category icons are displayed alongside category names
