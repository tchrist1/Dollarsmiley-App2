================================================================================
                            CONFIGURATION SYSTEM
================================================================================

DEBUG FLAG:
-----------
Location: constants/debug.ts

export const DEBUG_CONFIG = {
  ENABLE_DEBUG_MENU: true,  // Set to false to disable
  SHOW_DEBUG_INFO: false,
  VERBOSE_LOGGING: false,
};

ENABLE/DISABLE:
---------------
Method 1 (Recommended):
  - Set ENABLE_DEBUG_MENU: false in constants/debug.ts
  - Menu disappears completely

Method 2 (Automatic):
  - Menu only appears when __DEV__ is true
  - Production builds automatically hide it
  - No manual action needed

SAFETY CHECK:
-------------
export const shouldShowDebugMenu = (): boolean => {
  return DEBUG_CONFIG.ENABLE_DEBUG_MENU && __DEV__;
};

Two conditions must be true:
1. Flag is enabled
2. Development mode is active

================================================================================
                            PERMISSION SYSTEM
================================================================================

SCREEN ACCESS RULES:
--------------------

interface NavigationItem {
  label: string;
  route: string;
  icon: any;
  category: string;
  requiresAuth?: boolean;      // Login required
  requiresProvider?: boolean;  // Provider/Hybrid only
  requiresAdmin?: boolean;     // Admin only
  description?: string;
}

FILTERING LOGIC:
----------------
- Screens without authentication → Always visible
- requiresAuth → Only if user logged in
- requiresProvider → Only if Provider or Hybrid user
- requiresAdmin → Only if Admin user

Automatically filters based on current user profile.

================================================================================
                            PRODUCTION DEPLOYMENT
================================================================================

CRITICAL: DISABLE BEFORE PRODUCTION
------------------------------------

Step 1: Set Flag
  File: constants/debug.ts
  Change: ENABLE_DEBUG_MENU: false

Step 2: Verify
  - Build production version
  - Check burger menu not visible
  - Test bottom nav still works

Step 3: Build Commands
  $ npm run build:web
  $ expo build:ios
  $ expo build:android

AUTOMATIC PROTECTION:
---------------------
Even if flag is true, menu won't appear in production because:
- __DEV__ is false in release builds
- shouldShowDebugMenu() returns false
- Tab is not rendered

DOUBLE SAFETY:
--------------
1. Manual flag (ENABLE_DEBUG_MENU)
2. Automatic check (__DEV__)

Both must be true for menu to appear.

================================================================================
                            TROUBLESHOOTING
================================================================================

ISSUE: Menu Not Showing
-----------------------
Solution:
1. Check DEBUG_CONFIG.ENABLE_DEBUG_MENU is true
2. Verify running in development mode
3. Check console for errors
4. Restart development server

ISSUE: Navigation Not Working
------------------------------
Solution:
1. Verify route path is correct
2. Check screen file exists
3. Ensure proper default export
4. Look for typos in route

ISSUE: Screen Not Listed
-------------------------
Solution:
1. Add to navigationItems array
2. Set correct permission flags
3. Choose existing category
4. Import required icon

ISSUE: Search Not Finding
--------------------------
Solution:
1. Try different keywords
2. Check screen description
3. Browse categories instead
4. Verify spelling
